# Plan an academic week

Create a realistic plan for classes, assignments, study, and project work

Version 0.1

## Before starting

- Confirm the workflow trigger has occurred.
- Gather the named inputs in the approved workspace.
- Confirm the desired output and definition of done.
- Name the person responsible for every human review.
- Start a timer for each step.

## Run sequence

1. **Collect classes, commitments, and deadlines** — produce Complete weekly intake; target 15 minutes.
2. **Identify assignments** — produce Assignment and commitment list; target 5 minutes.
3. **Break large assignments into smaller tasks** — produce Actionable task list; target 10 minutes.
4. **Estimate effort** — produce Ranged effort estimates; target 8 minutes.
5. **Identify dependencies** — produce Dependency-aware task list; target 5 minutes.
6. **Prioritize work** — produce Prioritized task list; target 8 minutes.
7. **Create a realistic weekly schedule** — produce Weekly schedule; target 12 minutes.
8. **Create daily starting actions** — produce Daily kickoff list; target 5 minutes.
9. **Review unfinished work** — produce Recovery decisions; target 7 minutes.
10. **Update the following week** — produce Updated planning baseline; target 5 minutes.

## Human checkpoints

- **Collect classes, commitments, and deadlines:** The student confirms dates against official course sources.
- **Identify assignments:** Confirm every assignment against its official instructions.
- **Break large assignments into smaller tasks:** The student verifies that every rubric requirement is covered.
- **Estimate effort:** Adjust estimates for personal pace and current capacity.
- **Identify dependencies:** Confirm external dependencies and lead times.
- **Prioritize work:** The student makes the final priority decisions.
- **Create a realistic weekly schedule:** The student checks energy, accessibility needs, and feasibility.
- **Create daily starting actions:** Confirm each action matches the actual assignment.
- **Review unfinished work:** The student chooses a realistic recovery action.
- **Update the following week:** Confirm that completed or obsolete work is not carried forward.

## Stop and escalate

When a failure condition is met, do not improvise around missing evidence, privacy restrictions, or accountable decisions. Return to the named input owner or reviewer and record the interruption for the next workflow revision.
