# Professor lecture preparation

Trigger: the previous class ends. Output: an accuracy-reviewed teaching package plus a post-class reflection.

The complete redesigned sequence is generated from `lectureIntake` and contains: standardized workspace; source collection; learning objectives; lecture brief; lesson structure; slide plan; activity or discussion; student-level adaptation; academic review; final export; and post-class reflection.

Required assets include the session folder structure, lecture brief, learning objectives, slide plan, class activity, student-level adaptation checklist, academic review checklist, post-class reflection, and source-grounded AI instruction.

Boundaries: the professor retains teaching decisions and final approval; academic content requires review; student information is excluded; grading is not automated.
